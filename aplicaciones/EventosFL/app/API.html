Nodos

  1. En tanto que admin puede eliminar eventos para mantener una lista de eventos actualizada.

    • URL: '/nodos/ABorrarEvento'
    • Método: GET
    • Formato de salida: JSON
    • Roles: admin
    • Requisitos: No disponible

    • Cursos posibles

      1. Precondición: Datos del evento válidos.

        Postcondicion: Evento eliminado.

        resultado:

        • label:'/VEventos',
        • msg:"Evento borrado".

      2. Precondición: Datos del evento inválidos.

        Postcondicion: No ocurre nada.

        resultado:

        • label:'/VEvento',
        • msg:"Error al borrar evento".

    • URL: '/nodos/VEventos'
    • Método: GET
    • Formato de salida: JSON
    • Roles: usuario

  3. En tanto que admin puede crear eventos para mantener una lista de eventos actualizada.

    • URL: '/nodos/ACrearEvento'
    • Método: POST
    • Formato de entrada y salida: JSON
    • Parámetros POST:
      • nombre - Requerido.
      • descripcion - Opcional.
      • fecha - Requerido.
      • lugar - Requerido.
      • capacidad - Requerido.
      • afiche - Opcional.
    • Roles: admin
    • Requisitos: No disponible

    • Cursos posibles

      1. Precondición: Datos de evento válidos.

        Postcondicion: Evento creado exitosamente.

        resultado:

        • label:'/VEventos',
        • msg:"Evento creado".

      2. Precondición: Datos de evento inválidos.

        Postcondicion: No ocurre nada.

        resultado:

        • label:'/VCrearEvento',
        • msg:"Error al crear evento".

    • URL: '/nodos/VActEvento'
    • Método: GET
    • Formato de salida: JSON
    • Roles: admin

  5. En tanto que cualquier persona puede registrarse como usuario para utilizar la aplicación.

    • URL: '/nodos/ARegistrarUsuario'
    • Método: POST
    • Formato de entrada y salida: JSON
    • Parámetros POST:
      • username - Requerido. Long. mínima: 4. Long. máxima: 20.
      • password - Requerido. Long. mínima: 6. Long. máxima: 10.
      • nombre - Requerido.
      • apellido - Requerido.
      • email - Requerido.
      • isAdmin - Requerido.
    • Roles: usuario
    • Requisitos: No disponible

    • Cursos posibles

      1. Precondición: Datos del usuario válidos.

        Postcondicion: Usuario registrado exitosamente.

        resultado:

        • label:'/VInicioSesion',
        • msg:"Usuario registrado".

      2. Precondición: Datos del usuario inválidos.

        Postcondicion: No ocurre nada.

        resultado:

        • label:'/VRegistroUsuario',
        • msg:"Error al registrar usuario".

    • URL: '/nodos/VCrearEvento'
    • Método: GET
    • Formato de salida: JSON
    • Roles: admin

  7. En tanto que cualquier persona registrada puede iniciar sesión como usuario para utilizar la aplicación.

    • URL: '/nodos/AIniciarSesion'
    • Método: POST
    • Formato de entrada y salida: JSON
    • Parámetros POST:
      • username - Requerido. Long. mínima: 4. Long. máxima: 20.
      • password - Requerido. Long. mínima: 6. Long. máxima: 10.
    • Roles: Sin restricción de rol
    • Requisitos: No disponible

    • Cursos posibles

      1. Precondición: Usuario válido.

        Postcondicion: Sesión iniciada.

        resultado:

        • label:'/VEventos',
        • msg:"Sesión iniciada".

        Se continúa con el rol de usuario.

      2. Precondición: Error en datos ingresados.

        Postcondicion: No pasa nada.

        resultado:

        • label:'/VInicioSesion',
        • msg:"Error al iniciar sesión".

    • URL: '/nodos/VParticipante'
    • Método: GET
    • Formato de salida: JSON
    • Roles: usuario
